# 工资计算工作流使用说明

## 文件结构

```
salary-calculator/
├── run_salary.py      # 主程序
├── output/            # 输出目录（自动创建）
└── README.md          # 本说明文件
```

## 使用方法

### 1. 准备文件

将以下三个文件放到桌面（或修改 `run_salary.py` 中的路径）：

1. **总文件**：`202605--揽投人员基础数据表.xls`（或类似命名）
2. **自有揽投员核算表标快**：`2026年5月自有揽投员核算表标快.xlsx`
3. **外包人员标快**：`平舆外包人员标快2026年5月计件薪酬核算表.xlsx`

### 2. 修改配置（每月必做）

编辑 `run_salary.py`，找到 `CONFIG` 配置块，修改三个文件路径：

```python
CONFIG = {
    "total_file": r"C:\Users\yuson\Desktop\202605--揽投人员基础数据表.xls",  # 改这里
    "own_file": r"C:\Users\yuson\Desktop\2026年5月自有揽投员核算表标快.xlsx",  # 改这里
    "outsource_file": r"C:\Users\yuson\Desktop\平舆外包人员标快2026年5月计件薪酬核算表.xlsx",  # 改这里
    "output_dir": r"C:\Users\yuson\.openclaw\workspace\salary-calculator\output",  # 输出目录一般不改
    ...
}
```

### 3. 运行脚本

```bash
cd C:\Users\yuson\.openclaw\workspace\salary-calculator
python run_salary.py
```

### 4. 查看结果

输出文件在 `output/` 目录下：

- `2026年5月自有揽投员核算表标快.xlsx` — 自有人员表（填充了从总文件匹配的数据）
- `平舆外包人员标快2026年5月计件薪酬核算表.xlsx` — 外包人员表（填充了从总文件匹配的数据）
- 如果有"其他外包"人员，会在外包表中新增一个"其他外包"sheet

### 5. 验证数据

脚本会打印汇总信息：

- 平舆总人数
- 自有人员数
- 外包人员数（已有sheet）
- 其他外包人员数
- 营业员数

检查人数是否合理，然后打开输出Excel文件验证数据正确性。

## 重要说明

### 列映射规则

脚本使用硬编码的列映射规则解决总文件和目标表结构差异：

- **col 7-31**：1:1映射（退换货 + 自助计件 + 特快业务）
- **col 32-34**：电商业务量（1:1映射）
- **col 35**：总文件独有列"散户:结算差额率30%以上"，**跳过不粘贴**
- **col 36-41**：电商业务收入，映射到目标表 col 35-40

这个规则写在 `COLUMN_MAPPING_RULES` 字典中。如果以后文件结构变化，修改这个字典即可。

### 自动识别人员类别

- **自有人员**：自有揽投员核算表标快中的姓名
- **外包人员**：外包人员标快中的姓名
- **营业员**：平舆全部人员 - 自有 - 外包 = 营业员（自动计算）
- **其他外包**：外包人员中不在已有外包sheet中的，自动归入新sheet

### 处理未匹配人员

如果自有/外包表中有人员不在总文件中（如当月无业务数据），脚本会打印警告但继续运行。

## 常见问题

**Q: 总文件中找不到某些人怎么办？**  
A: 这是正常的，可能当月没有业务数据。脚本会跳过这些人，不影响其他人的数据填充。

**Q: 如何验证数据正确？**  
A: 在输出Excel中随机抽查几个人的数据，对比总文件中的原始数据是否一致。特别注意电商业务收入部分（col 35-41）。

**Q: 文件结构变化怎么办？**  
A: 修改 `CONFIG` 中的行列参数（如 `name_col`、`data_start_row` 等），或联系开发者更新脚本。

**Q: 能否批量处理多个月份？**  
A: 目前不支持。每月需手动修改配置路径后运行一次。
