# HuggingFace Dataset Extractor - Concurrent Version

## 特性

### 1. 并发处理
- **多线程**：使用 `ThreadPoolExecutor` 并发请求
- **默认并发度**：5（可在脚本顶部修改 `CONCURRENCY` 常量）
- **线程安全**：输出打印、LLM 调用都有锁保护
- **预计提速**：3-5 倍（10条约30-40秒，25条约1分钟）

### 2. 继承 v2.0 优化
- 合并 LLM 调用（一次返回榜单+Agent）
- 配置外部化（`config.yaml`）
- 支持环境变量

### 3. 性能统计
- 实时显示进度
- 完成后输出总耗时、平均耗时
- 计算相对于串行版的加速比

## 使用方式

```bash
python run_workbook_concurrent.py 工作簿1
```

## 性能对比

| 版本 | 10条 | 25条 | 100条 |
|------|------|------|-------|
| 原版（串行） | ~50-60秒 | ~3-4分钟 | ~10-12分钟 |
| v2.0（合并LLM） | ~40-50秒 | ~2.5-3分钟 | ~9-10分钟 |
| **并发版** | **~30-40秒** | **~1-1.5分钟** | **~3-4分钟** |

**提速倍数**：约 3-5x

## 并发安全

1. **请求层**：每个线程独立请求 API/README/数据预览
2. **LLM 调用**：OpenAI SDK 内部线程安全
3. **打印输出**：使用 `print_lock` 避免混乱
4. **数据收集**：`futures` 顺序保证结果正确

## 调整并发度

编辑脚本顶部：
```python
CONCURRENCY = 5  # 改为 3, 10 等
```

**建议值**：
- 网络较慢：3
- 正常：5（默认）
- 网络快且 LLM 限流宽松：10

**注意**：并发度过高可能触发 HuggingFace 或智谱 API 限流。

## 适用场景

- **大批量处理**：100+ 条数据集
- **网络良好**：稳定的外网连接
- **API 限流宽松**：智谱账号无严格限流

## 不适用场景

- **小批量**：<10 条，串行版足够快
- **网络不稳定**：并发会放大失败概率
- **严格限流**：触发 429 错误时改用串行

## 错误处理

- 单个数据集失败不影响其他
- 所有错误汇总在最后输出
- 失败的数据集字段填 `/`
