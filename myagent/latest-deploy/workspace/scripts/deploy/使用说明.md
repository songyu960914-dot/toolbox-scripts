# OpenClaw 部署工具使用说明

## 目的

在新电脑上一键部署你的 AI 助手，包含所有记忆、配置和工作脚本。

---

## 步骤 1：打包（在当前电脑上）

在当前电脑上运行：

```bash
python scripts/package_deploy.py
```

这会在桌面生成 `openclaw-deploy-YYYYMMDD.zip`，包含：
- OpenClaw 配置
- 所有记忆文件（MEMORY.md, daily-logs 等）
- 灵魂/身份配置（SOUL.md, IDENTITY.md 等）
- 工作脚本（safe_delete, HuggingFace 提取工具等）
- 自动安装脚本

---

## 步骤 2：传输到新电脑

将 `openclaw-deploy-YYYYMMDD.zip` 拷贝到新电脑（U盘/网盘/邮件均可）。

---

## 步骤 3：部署（在新电脑上）

1. **解压缩**
   ```
   右键 → 解压到当前文件夹
   ```

2. **以管理员身份运行 PowerShell**
   ```
   Win + X → Windows PowerShell (管理员)
   ```

3. **进入解压目录**
   ```powershell
   cd C:\Users\你的用户名\Desktop\openclaw-deploy-YYYYMMDD
   ```

4. **执行安装脚本**
   ```powershell
   powershell -ExecutionPolicy Bypass -File deploy.ps1
   ```

脚本会自动：
- 检查/安装 Node.js, Python, GitHub CLI
- 安装 OpenClaw 和 Python 依赖
- 部署所有配置和记忆文件

---

## 步骤 4：手动配置

安装完成后，需要手动配置以下内容：

### 4.1 GitHub 认证
```bash
gh auth login
```
按提示选择：GitHub.com → HTTPS → Yes → Login with a web browser

### 4.2 LiteLLM 配置（如果使用本地代理）
编辑 `C:\Users\你的用户名\.openclaw\openclaw.json`

找到 `models.providers.litellm` 部分，修改：
```json
"baseUrl": "http://127.0.0.1:4000",
"apiKey": "你的Key"
```

### 4.3 智谱 API Key
编辑 `C:\Users\你的用户名\.openclaw\workspace\huggingface-dataset-extractor\*\config.yaml`

将占位符 `${ZHIPU_API_KEY}` 改为真实 Key：
```yaml
api_key: "你的智谱APIKey"
```

### 4.4 Git 代理（中国网络）
如果使用 Clash 等代理：
```bash
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

---

## 步骤 5：启动

```bash
openclaw start
```

打开浏览器访问：http://localhost:18789

你的 AI 助手已完整恢复，包含所有记忆。

---

## 注意事项

1. **API Keys 不在部署包中**
   - LiteLLM API Key（如果用本地代理）
   - 智谱 API Key
   需要手动配置

2. **GitHub 凭据不迁移**
   需要在新电脑重新 `gh auth login`

3. **记忆数据包含隐私信息**
   部署包包含你的工作记录和配置，请妥善保管

4. **定期更新部署包**
   记忆会随时间增长，建议每月重新打包一次

---

## 故障排查

### 问题：无法安装 Node.js/Python
- 手动下载安装：
  - Node.js: https://nodejs.org/
  - Python: https://www.python.org/

### 问题：OpenClaw 启动失败
- 检查端口占用：`netstat -ano | findstr 18789`
- 查看日志：`C:\Users\你的用户名\.openclaw\logs\`

### 问题：GitHub 认证失败
- 使用 token 模式：`gh auth login --with-token`
- 从 https://github.com/settings/tokens 生成 token
