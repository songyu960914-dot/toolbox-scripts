## 场景1：新用户首次部署

**适用于：** 从未用过 OpenClaw，想从模板开始配置。

### 步骤 1：安装前置依赖

#### Windows

```powershell
# 安装 Node.js（推荐 LTS 版本）
winget install OpenJS.NodeJS.LTS

# 安装 Python 3.12
winget install Python.Python.3.12

# 安装 GitHub CLI
winget install GitHub.cli
```

安装后重启终端。

#### macOS

```bash
# 使用 Homebrew
brew install node python@3.12 gh
```

#### Linux (Ubuntu/Debian)

```bash
# Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Python
sudo apt-get install -y python3.12 python3-pip

# GitHub CLI
curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null
sudo apt update
sudo apt install gh
```

### 步骤 2：安装 OpenClaw

```bash
npm install -g openclaw
```

验证安装：
```bash
openclaw --version
```

### 步骤 3：克隆配置模板

```bash
# 克隆仓库
git clone https://github.com/songyu960914-dot/toolbox-scripts.git
cd toolbox-scripts/myagent

# 复制配置模板
mkdir -p ~/.openclaw/workspace
cp -r config-template/* ~/.openclaw/workspace/
```

Windows PowerShell：
```powershell
mkdir $env:USERPROFILE\.openclaw\workspace -Force
Copy-Item -Recurse config-template\* $env:USERPROFILE\.openclaw\workspace\
```

### 步骤 4：个性化配置

#### 4.1 编辑身份信息

```bash
cd ~/.openclaw/workspace
```

编辑 `IDENTITY.md`：
```markdown
- **Name:** 给你的助手起个名字
- **Creature:** AI? 机器人? 精灵?
- **Vibe:** 性格风格
- **Emoji:** 签名表情
```

编辑 `USER.md`：
```markdown
- **Name:** 你的名字
- **What to call them:** 助手怎么称呼你
- **Timezone:** 你的时区（如 Asia/Shanghai）
```

#### 4.2 调整个性（可选）

编辑 `SOUL.md` 自定义助手的：
- 说话风格
- 价值观
- 工作方式

### 步骤 5：配置认证

#### GitHub（推荐）

```bash
gh auth login
```

选择：
- **What account?** → GitHub.com
- **Protocol?** → HTTPS
- **Authenticate Git?** → Yes
- **How?** → Login with a web browser

#### LiteLLM（如果使用本地代理）

编辑 `~/.openclaw/openclaw.json`：

```json
{
  "models": {
    "providers": {
      "litellm": {
        "baseUrl": "http://127.0.0.1:4000",
        "apiKey": "your-litellm-key"
      }
    }
  }
}
```

### 步骤 6：启动

```bash
openclaw start
```

打开浏览器访问：http://localhost:18789

**首次对话：**
```
你：嘿，你是谁？
助手：（会按照 IDENTITY.md 回答）
```

## 场景2：现有用户迁移到新电脑

**适用于：** 已经在旧电脑配置好，想完整迁移到新电脑。

### A. 在旧电脑上打包

#### 步骤 A1：安装 Python 依赖

```bash
pip install pandas openpyxl requests openai pyyaml
```

#### 步骤 A2：运行打包脚本

```bash
cd ~/.openclaw/workspace/scripts
python package_deploy.py
```

会在桌面生成 `openclaw-deploy-YYYYMMDD.zip`，包含：
- ✅ 所有记忆（MEMORY.md + daily-logs）
- ✅ 所有配置（AGENTS, SOUL, IDENTITY 等）
- ✅ 所有工作脚本
- ✅ OpenClaw 配置（openclaw.json）
- ✅ 自动部署脚本

#### 步骤 A3：传输到新电脑

用 U盘、网盘、邮件等方式将 zip 文件拷贝到新电脑。

---

### B. 在新电脑上部署

#### 步骤 B1：解压

```
右键 → 解压到当前文件夹
```

得到 `openclaw-deploy-YYYYMMDD/` 目录。

#### 步骤 B2：以管理员身份运行部署脚本

**Windows：**

1. **Win + X** → **Windows PowerShell (管理员)**
2. 进入解压目录：
   ```powershell
   cd C:\Users\你的用户名\Desktop\openclaw-deploy-YYYYMMDD
   ```
3. 执行部署：
   ```powershell
   powershell -ExecutionPolicy Bypass -File deploy.ps1
   ```

脚本会自动：
- ✅ 检查并安装 Node.js, Python, GitHub CLI
- ✅ 安装 OpenClaw
- ✅ 安装 Python 依赖（pandas, requests 等）
- ✅ 部署所有配置到 `~/.openclaw/workspace/`
- ✅ 恢复 openclaw.json

**macOS/Linux：**

（暂不支持自动部署脚本，需手动操作）

```bash
# 1. 安装依赖（参考场景1）
# 2. 手动复制文件
cp -r openclaw-deploy-YYYYMMDD/workspace/* ~/.openclaw/workspace/
cp openclaw-deploy-YYYYMMDD/config/openclaw.json ~/.openclaw/
```

#### 步骤 B3：手动配置（必须）

部署脚本完成后，还需手动配置以下内容：

##### 3.1 GitHub 认证

```bash
gh auth login
```

##### 3.2 检查 API Keys

**LiteLLM（如果使用）：**
```bash
# 编辑（Windows）
notepad %USERPROFILE%\.openclaw\openclaw.json

# 编辑（macOS/Linux）
nano ~/.openclaw/openclaw.json
```

找到 `models.providers.litellm.apiKey`，确认是否正确。

**智谱 API（如果使用 HuggingFace 工具）：**
```bash
# Windows
notepad %USERPROFILE%\.openclaw\workspace\huggingface-dataset-extractor\huggingface-dataset-extractor_v2\config.yaml

# macOS/Linux
nano ~/.openclaw/workspace/huggingface-dataset-extractor/huggingface-dataset-extractor_v2/config.yaml
```

将 `${ZHIPU_API_KEY}` 改为真实 Key。

##### 3.3 配置 Git 代理（中国网络）

如果使用 Clash 等代理：

```bash
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

#### 步骤 B4：启动并验证

```bash
openclaw start
```

打开浏览器访问：http://localhost:18789

**验证记忆：**
```
你：你还记得昨天我们做了什么吗？
助手：（会从 MEMORY.md 和 daily-logs 回答）
```

## 场景3：多台电脑同步配置

**适用于：** 家里/公司多台电脑，想保持配置一致。

### 方案 A：通过 Git 仓库同步

#### 初始设置（第一台电脑）

```bash
cd ~/.openclaw/workspace

# 初始化仓库
git init
git add AGENTS.md SOUL.md IDENTITY.md USER.md MEMORY.md TOOLS.md HEARTBEAT.md scripts/
git commit -m "Initial workspace config"

# 推送到私有仓库
git remote add origin https://github.com/你的用户名/my-openclaw-config.git
git push -u origin main
```

**⚠️ 注意：** 使用**私有仓库**，因为包含你的记忆和配置。

#### 在其他电脑上克隆

```bash
# 安装 OpenClaw（参考场景1）
npm install -g openclaw

# 克隆配置
git clone https://github.com/你的用户名/my-openclaw-config.git ~/.openclaw/workspace

# 启动
openclaw start
```

#### 日常同步

**电脑 A 做了修改：**
```bash
cd ~/.openclaw/workspace
git add -A
git commit -m "Update memory and tools"
git push
```

**电脑 B 同步：**
```bash
cd ~/.openclaw/workspace
git pull
openclaw restart
```

### 方案 B：定期打包迁移

适合不想用 Git 的场景，定期运行打包脚本，手动拷贝。

## 进阶配置

### 自定义 Heartbeat 任务

编辑 `~/.openclaw/workspace/HEARTBEAT.md`：

```markdown
## 定期任务

### 清理 temp/ 文件夹
检查 `workspace/temp/` 目录，删除超过 3 天的文件。

### 清理中转站
检查 `workspace/trash/` 目录，彻底删除超过 3 天的压缩文件。

### 自定义任务（示例）
检查某个 API 的状态，如果异常则通知。
```

修改后重启 OpenClaw。

### 添加自定义工具脚本

1. 在 `~/.openclaw/workspace/scripts/` 放置新脚本
2. 在 `TOOLS.md` 中记录使用方法：

```markdown
## 我的自定义工具

**使用方法：**
```bash
python scripts/my_tool.py --arg value
```

**说明：** 这个工具做什么...
```

3. 重启 OpenClaw

### 配置多个 Agent

编辑 `~/.openclaw/openclaw.json`：

```json
{
  "agents": {
    "list": [
      {
        "id": "main",
        "model": { "primary": "litellm/claude-opus-4-6" }
      },
      {
        "id": "coding",
        "model": { "primary": "litellm/claude-sonnet-4-6" },
        "workspace": "/path/to/coding-workspace"
      }
    ]
  }
}
```

## 常见问题

### Q1: 部署脚本卡住不动

**原因：** 可能在下载依赖。

**解决：**
- 检查网络连接
- 如果在中国，配置 npm 镜像：
  ```bash
  npm config set registry https://registry.npmmirror.com
  ```

### Q2: OpenClaw 启动后浏览器打不开

**检查：**
```bash
# 端口是否被占用
netstat -ano | findstr 18789  # Windows
lsof -i :18789                # macOS/Linux
```

**解决：**
- 修改 `~/.openclaw/openclaw.json` 中的 `gateway.port`
- 或者杀掉占用进程

### Q3: 助手没有记忆

**原因：** MEMORY.md 或 daily-logs 没有正确部署。

**检查：**
```bash
# Windows
dir %USERPROFILE%\.openclaw\workspace\MEMORY.md

# macOS/Linux
ls -la ~/.openclaw/workspace/MEMORY.md
```

**解决：**
- 确认文件存在
- 重启 OpenClaw：`openclaw restart`

### Q4: GitHub 认证失败

**解决：**
```bash
# 使用 token 模式
gh auth login --with-token

# 从 https://github.com/settings/tokens 生成 token
# 粘贴 token 到终端
```

### Q5: Python 脚本报错找不到模块

**解决：**
```bash
# 重新安装依赖
pip install --user pandas openpyxl requests openai pyyaml
```

### Q6: 部署包太大

**原因：** 包含了大量 daily-logs。

**解决：**
- 编辑 `scripts/package_deploy.py`
- 修改保留天数：
  ```python
  cutoff = datetime.now() - timedelta(days=7)  # 改为7天
  ```

### Q7: 迁移后助手"失忆"了

**原因：** MEMORY.md 或 daily-logs 没有包含在部署包中。

**检查：**
```bash
# 查看部署包内容
unzip -l openclaw-deploy-YYYYMMDD.zip | grep MEMORY
```

**解决：**
- 手动从旧电脑复制：
  ```bash
  # 旧电脑
  scp ~/.openclaw/workspace/MEMORY.md user@新电脑:~/.openclaw/workspace/
  ```

## 性能优化

### 减少 LLM 调用成本

编辑 `AGENTS.md`，调整思考模式：

```markdown
<thinking_mode>adaptive</thinking_mode>
<thinking_effort>low</thinking_effort>  # 降低推理成本
```

### 加速启动

关闭不需要的功能，编辑 `openclaw.json`：

```json
{
  "tools": {
    "web": {
      "search": { "enabled": false },
      "fetch": { "enabled": false }
    }
  }
}
```

## 卸载

### 完全卸载 OpenClaw

```bash
# 卸载 OpenClaw
npm uninstall -g openclaw

# 删除配置和数据
rm -rf ~/.openclaw  # macOS/Linux
rmdir /s %USERPROFILE%\.openclaw  # Windows
```

### 仅删除配置保留数据

```bash
# 备份
cp -r ~/.openclaw ~/.openclaw.backup

# 删除配置
rm ~/.openclaw/openclaw.json

# 重新初始化
openclaw start
```
